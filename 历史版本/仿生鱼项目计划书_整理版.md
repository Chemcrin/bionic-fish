# 🐟 仿生鱼项目计划书

> **版本：** v1.0 ｜ **日期：** 2026-05-08 ｜ **状态：** 初版整理

---

## 📌 一、项目定位

制作一只以**稳定性和可控性**为优先目标的仿生鱼，用于水下任务演示或比赛。要求：

- 水中稳定游动
- 可控沉浮
- 通过尾鳍、背鳍、臀鳍配合完成方向控制
- 实现对不同高度、不同位置坐标点的接近与打卡

**首版策略：** 先做可靠线控原型 → 再决定是否加入视觉识别与自主决策

---

## 🎯 二、项目目标

### 2.1 核心目标

| 序号 | 目标 | 说明 |
|:---:|------|------|
| 1 | 制作可下水实体样机 | 能稳定运行 |
| 2 | 基础推进 | 尾部摆动驱动鱼体前进 |
| 3 | 方向控制 | 尾鳍/背鳍/臀鳍调整方向与姿态 |
| 4 | 可控沉浮 | 上浮、下潜、定深控制 |
| 5 | 任务打卡 | 到达不同坐标点并完成触发/接近/识别 |
| 6 | 水下安全 | 电路、舵机、电池、主控区域可靠防水 |

### 2.2 首版边界

**采用「线控 + 手动/半自动控制」方式验证原型。**
水下无线电波通信衰减明显，首版不建议将自主视觉寻点作为必须完成项。

后期增强版可加入摄像头、视觉识别模块、更强算力主控或协处理器，实现自动寻点、自动接近和自动打卡。

---

## 🔄 三、典型任务流程

```
下水前检查（防水/电量/舵机回中/通信/传感器）  
       ↓  
入水姿态稳定（无侧翻、漏水、失控）  
       ↓  
操作者控制前进、转向、上浮、下潜  
       ↓  
接近指定坐标点 → 调整深度  
       ↓  
到达后完成打卡动作（触碰/停留/识别/触发传感器）  
       ↓  
切换至下一个目标点  
       ↓  
任务结束 → 返航或人工回收 → 记录数据
```

## 📋 四、需求拆解

### 4.1 功能需求

| 编号 | 需求 | 优先级 | 说明 | 验收方式 |
|:----:|------|:------:|------|----------|
| F01 | 水中稳定漂浮与姿态保持 | **P0** | 入水后不应明显侧翻、头重脚轻或持续翻滚 | 静水中观察姿态，记录横滚和俯仰偏差 |
| F02 | 尾鳍推进 | **P0** | 尾部舵机驱动尾鳍周期摆动，提供前进动力 | 在水槽或泳池中完成连续前进 |
| F03 | 方向控制 | **P0** | 通过尾鳍、背鳍、臀鳍控制游动方向 | 能完成左转、右转、修正航向 |
| F04 | 可控沉浮 | **P0** | 根据目标点高度完成上浮、下潜或定深 | 在不同深度位置完成稳定运动 |
| F05 | 坐标点打卡 | **P0** | 能接近或触发比赛要求的坐标点 | 按比赛规则完成打卡动作 |
| F06 | 线控通信 | **P0** | 首版通过线缆传输控制指令或调试数据 | 水下运行期间指令响应稳定 |
| F07 | 舵机与鳍片联动 | **P0** | 舵机带动鳍片在限定角度范围内顺畅摆动 | 舵机无卡滞、无过载发热 |
| F08 | 防水与漏电保护 | **P0** | 主控、电池、舵机接线处需要密封 | 浸水测试后内部无水、无短路 |
| F09 | 配重与重心调节 | **P0** | 内部设置可调配重，降低重心并改善平衡 | 调整配重后姿态明显改善 |
| F10 | 视觉自主寻点 | **P2** | 增强版能力，需要摄像头和更强算力 | 完成目标识别和自主接近测试 |

### 4.2 非功能需求

| 类型 | 要求 | 说明 |
|------|------|------|
| **稳定性** | 首版优先稳定，不追求高速 | 低重心常规鱼型，避免过窄、过高、重心偏上结构 |
| **可维护性** | 舵机、电池、主控、配重方便拆装 | 预留检修口，避免每次维护破坏密封结构 |
| **防水性** | 电控舱、线缆出口、舵机区域重点防水 | 密封圈、密封胶、轴封、防水接头或独立密封舱 |
| **可制造性** | 外壳使用 3D 打印 PETG 材质 | 兼顾强度、外观、打印难度和后处理防水 |
| **可调试性** | 保留日志、串口或上位机调试接口 | 便于定位舵机、姿态、深度和通信问题 |
| **安全性** | 电池、电路和舵机过流需要保护 | 防止水下短路、发热和失控 |

## 🏗️ 五、总体技术路线

### 阶段 A：可靠线控原型

目标是尽快做出可下水、可游、可控方向与深度的样机。

```
主控: STM32
舵机: 防水舵机驱动尾鳍/背鳍/臀鳍
通信: 线缆控制与调试（降低水下通信风险）
外壳: PETG 3D 打印
配重: 配重 + 浮力材料调节重心与浮心
控制: 先完成人工控制打卡，再优化控制算法
```

### 阶段 B：自主增强版本

在阶段 A 稳定后加入视觉/传感器自主能力。

```
摄像头/视觉模块
更强算力 MCU 或协处理器
目标识别、自动寻点、自动打卡
深度闭环、姿态闭环、任务状态机
```

## 🧩 七、模块设计要求

### 7.1 机械结构

| 模块 | 设计要求 | 需要完成的工作 |
|------|----------|---------------|
| **鱼体外壳** | 采用鲽鱼/鳐鱼等低重心稳定构型；PETG 3D 打印 | CAD 外形，确定长宽高、壁厚、检修口位置 |
| **尾鳍机构** | 舵机驱动尾部周期摆动，提供主要推进力 | 设计舵机座、尾柄、连杆/曲柄摇杆机构，验证摆角范围 |
| **背鳍/臀鳍** | 用于方向修正、姿态稳定和辅助沉浮 | 确定鳍片位置、面积、舵机安装方式和最大摆角 |
| **传动机构** | 可增加齿轮、连杆、曲柄摇杆等结构 | 做运动仿真或手动样件验证，避免卡滞和干涉 |
| **防水结构** | 主控、电池、线缆、舵机接线处重点密封 | 密封舱、密封圈槽、线缆防水接头、检修盖 |
| **配重结构** | 内部需要可调配重，降低重心并抑制游动摇晃 | 可移动配重槽，支持前后、左右、上下微调 |

### 7.2 电控系统

| 模块        | 建议方案                  | 说明                                 |
| --------- | --------------------- | ---------------------------------- |
| **主控**    | STM32F4 系列或同级         | 若只控制舵机，低端 STM32 也可；若后续加更多传感器建议预留性能 |
| **舵机**    | 防水舵机优先                | 若用普通舵机，应放入密封舱并处理输出轴密封              |
| **舵机驱动**  | PWM 控制或独立舵机驱动板        | 多舵机时建议独立供电，避免主控电源波动                |
| **电源**    | 锂电池 + 稳压模块 + 保险保护     | 舵机电源和主控电源建议分级设计                    |
| **姿态传感器** | IMU（如 JY61P）          | 检测俯仰、横滚、偏航趋势                       |
| **深度传感器** | 水压传感器                 | 定深和沉浮控制                            |
| **漏水检测**  | 导电式或湿度传感器             | 一旦检测漏水应停止舵机并提示回收                   |
| **通信**    | 线控 UART、RS485 或 CAN   | 具体协议根据线长、抗干扰和上位机方案确定               |


### 7.3 软件与控制

| 软件模块 | 功能 |
|----------|------|
| **舵机初始化** | 舵机回中、限位设置、运动范围标定 |
| **尾鳍摆动控制** | 周期性摆动信号，控制摆幅和频率 |
| **方向控制** | 将左转、右转、修正航向等指令转为舵机角度 |
| **沉浮控制** | 根据深度和姿态传感器控制鳍片角度或浮力调节机构 |
| **姿态稳定** | 根据 IMU 数据修正横滚和俯仰 |
| **通信协议** | 接收控制指令，返回状态、电量、深度、漏水等数据 |
| **安全保护** | 通信丢失、漏水、低电压或舵机异常时进入保护状态 |
| **任务状态机** | 下水、自检、巡游、接近目标、打卡、返航等状态切换 |

## 💧 八、沉浮控制方案选择

可控沉浮是项目关键难点之一。建议先明确比赛是否要求"原地升降/悬停"，再决定方案复杂度。

| 方案 | 优点 | 缺点 | 适用阶段 |
|------|------|------|:--------:|
| **中性浮力 + 鳍面控制升降** ✅ | 结构简单，适合首版快速实现 | 需要前进速度，低速/原地定深能力较弱 | **阶段 A** |
| **可移动配重** | 有助于调整俯仰和重心 | 响应较慢，机构占空间 | 阶段 A/B |
| **活塞式浮力舱** | 能实现更真实的沉浮控制 | 机械和防水复杂度高 | 阶段 B |
| **微型水泵调节浮力舱** | 可实现主动浮力变化 | 需处理水泵、管路、密封和响应速度 | 阶段 B |

**首版建议：** 先把鱼体调到接近中性浮力，再用鳍面控制上浮和下潜。如果比赛要求低速定深或原地悬停，再升级为主动浮力调节。

## ⚠️ 十一、关键风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 水下通信不稳定 | 无法可靠控制 | 首版线控；线长较长时优先 RS485 或 CAN |
| 防水失败 | 电路短路、样机损坏 | 分阶段浸水测试，电控舱独立密封，线缆出口重点处理 |
| 重心与浮心不匹配 | 鱼体侧翻或游动摇晃 | 可调配重，先静水调平再动态测试 |
| 舵机力矩不足 | 推进弱或鳍片卡滞 | 先做单鳍负载测试，预留更大扭矩舵机安装空间 |
| 传动机构卡滞 | 舵机发热、动作失控 | 简单可靠的连杆/曲柄机构，减少复杂齿轮级数 |
| PETG 外壳渗水 | 内部进水或结构变形 | 增加壁厚，优化打印参数，外表面做涂层或密封处理 |
| 沉浮控制不足 | 无法到达不同高度目标点 | 首版调中性浮力，必要时升级主动浮力舱 |
| 控制算法耦合 | 转向、俯仰、沉浮互相影响 | 先手动标定各鳍片作用，再逐步加入闭环控制 |

## ✅ 十二、验收标准

首版样机建议按以下标准验收：

- **防水：** 静置浸水 30 分钟后，电控舱和电池舱无明显进水
- **上电：** 入水前自检通过，舵机可回中，通信稳定
- **姿态：** 静水中鱼体保持基本水平，不持续侧翻
- **推进：** 尾鳍摆动后能连续前进，舵机无明显过热
- **转向：** 能完成可控左转、右转和航向修正
- **沉浮：** 能完成上浮、下潜或不同深度目标接近
- **打卡：** 能按比赛或演示规则完成至少一次目标点打卡
- **维护：** 电池、主控、配重可在不破坏主体结构的情况下拆装调整

## 📝 十三、接下来要做的事

### 立即确认的问题

- 比赛/展示场地水池尺寸、水深和水质环境
- 坐标点数量、位置、高度范围和打卡判定方式
- 是否允许线控？线缆是否可以传输电源，还是只能传信号
- 仿生鱼尺寸、重量、材料、电池和安全限制
- 是否必须自主寻点，还是允许人工控制完成任务
- 预算上限、制作周期和可用设备（3D 打印机、车床等）

### 一周内建议完成

- 输出《需求确认表》
- 选择首版鱼型，初步确定鱼体尺寸和舱内布局
- 确定控制方式（首版建议线控）
- 确定沉浮路线（首版中性浮力 + 鳍面控制）
- 绘制系统框图和机械布局草图
- 列出第一版采购清单

### 各组任务分工

| 组别 | 主要任务 |
|------|----------|
| **机械组** | 鱼体外壳 CAD、主控舱/电池舱/密封结构、鳍片与舵机座、传动机构、可调配重槽 → 3D 打印验证 |
| **电控组** | STM32 选型、舵机数量/电压/供电方案、接线图、IMU/深度/漏水传感器接入、电池保护设计 → 陆上联调 |
| **软件组** | STM32 固件工程、PWM 输出/回中/限位、尾鳍摆动控制、方向/沉浮控制、通信协议、姿态/深度闭环、任务状态机 |
| **测试组** | 防水测试流程、静水配重测试、舵机寿命测试、水中推进/转向/沉浮测试 → 问题记录与追踪 |
