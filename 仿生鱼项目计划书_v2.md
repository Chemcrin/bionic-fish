# 🐟 仿生鱼项目计划书 V2

> **版本：** v2.1（硬件选型确认版） | **日期：** 2026-05-24 | **状态：** 硬件选型确认完成

---

## 📌 一、项目定位

制作一只以**稳定性和可控性**为优先目标的仿生鱼，用于水下任务演示或比赛。要求：

- 水中稳定游动
- 可控沉浮
- 通过尾鳍、背鳍、臀鳍配合完成方向控制
- 实现对不同高度、不同位置坐标点的接近与打卡

**首版策略：** 先做可靠线控原型 → 再决定是否加入视觉识别与自主决策

---

## 🎯 二、项目目标

### 2.1 核心目标

| 序号 | 目标 | 说明 |
|:---:|------|------|
| 1 | 制作可下水实体样机 | 能稳定运行 |
| 2 | 基础推进 | 尾部摆动驱动鱼体前进 |
| 3 | 方向控制 | 尾鳍/背鳍/臀鳍调整方向与姿态 |
| 4 | 可控沉浮 | 上浮、下潜、定深控制 |
| 5 | 任务打卡 | 到达不同坐标点并完成触发/接近/识别 |
| 6 | 水下安全 | 电路、舵机、电池、主控区域可靠防水 |

### 2.2 首版边界

**采用「线控 + 手动/半自动控制」方式验证原型。**
水下电磁波通信衰减明显（2.4G 在水中几乎无法穿透），首版不建议将自主视觉寻点作为必须完成项。

**视觉方案当前阶段舍弃：** 水下视觉识别技术门槛高、对光照和水质敏感，且阶段A的首要目标是做出能稳定游动和可控沉浮的物理样机。将自主寻点留到阶段B增强版再考虑。

## 🔄 三、典型任务流程

```
下水前检查（防水/电量/舵机回中/通信/传感器）
       ↓
入水姿态稳定（无侧翻、漏水、失控）
       ↓
操作者控制前进、转向、上浮、下潜
       ↓
接近指定坐标点 → 调整深度
       ↓
到达后完成打卡动作（触碰/停留/识别/触发传感器）
       ↓
切换至下一个目标点
       ↓
任务结束 → 返航或人工回收 → 记录数据
```

### 4.2 非功能需求

| 类型 | 要求 | 说明 |
|------|------|------|
| **稳定性** | 首版优先稳定，不追求高速 | 低重心常规鱼型，避免过窄、过高、重心偏上结构 |
| **可维护性** | 舵机、电池、主控、配重方便拆装 | 预留检修口，避免每次维护破坏密封结构 |
| **防水性** | 电控舱、线缆出口、舵机区域重点防水 | 密封圈、密封胶、轴封、防水接头或独立密封舱 |
| **可制造性** | 外壳使用 3D 打印 PETG 材质 | 兼顾强度、外观、打印难度和后处理防水 |
| **可调试性** | 保留日志、串口或上位机调试接口 | 便于定位舵机、姿态、深度和通信问题 |
| **安全性** | 电池、电路和舵机过流需要保护 | 防止水下短路、发热和失控 |

## 🏗️ 五、总体技术路线

### 阶段 A：可靠线控原型

目标是尽快做出可下水、可游、可控方向与深度的样机。

```
主控: STM32F103ZET6（优先）/ C8T6（备选）
舵机: 防水舵机驱动尾鳍/背鳍/臀鳍
通信: 线缆控制与调试（降低水下通信风险）
外壳: PETG 3D 打印
配重: 配重 + 浮力材料调节重心与浮心
控制: 先完成人工控制打卡，再优化控制算法
```

### 阶段 B：自主增强版本

在阶段 A 稳定后加入增强能力。**当前阶段B暂不包含视觉识别方案**（水下电磁信号穿透困难，视觉当前使用场景有限）。

```
更强算力 MCU 或协处理器（ESP32-S3 等，按需引入）
目标识别、自动寻点、自动打卡（传感器方案，非视觉方案）
深度闭环、姿态闭环、任务状态机
```

## 🧩 七、模块设计要求

### 7.1 机械结构

| 模块 | 设计要求 | 需要完成的工作 |
|------|----------|---------------|
| **鱼体外壳** | 采用鲈鱼/鳊鱼等低重心稳定构型；PETG 3D 打印 | CAD 外形，确定长宽高、壁厚、检修口位置 |
| **尾鳍机构** | 舵机驱动尾部周期摆动，提供主要推进力 | 设计舵机座、尾柄、连杆/曲柄摇杆机构，验证摆角范围 |
| **背鳍/臀鳍** | 用于方向修正、姿态稳定和辅助沉浮 | 确定鳍片位置、面积、舵机安装方式和最大摆角 |
| **传动机构** | 可增加齿轮、连杆、曲柄摇杆等结构 | 做运动仿真或手动样件验证，避免卡滞和干涉 |
| **防水结构** | 主控、电池、线缆、舵机接线处重点密封 | 密封舱、密封圈槽、线缆防水接头、检修盖 |
| **配重结构** | 内部需要可调配重，降低重心并抑制游动摇晃 | 可移动配重槽，支持前后、左右、上下微调 |

### 7.2 电控系统

| 模块 | 建议方案 | 说明 |
| --------- | --------------------- | ---------------------------------- |
| **主控** | STM32F103ZET6（优先） | 见下方选型分析章节 |
| **舵机** | 防水舵机优先 | 若用普通舵机，应放入密封舱并处理输出轴密封 |
| **舵机驱动** | PWM 控制或独立舵机驱动板 | 多舵机时建议独立供电，避免主控电源波动 |
| **电源** | 锂电池 + 稳压模块 + 保险保护 | 舵机电源和主控电源建议分级设计 |
| **姿态传感器** | JY61P（推荐） | 直接输出欧拉角，串口读取即用，省去姿态解算开发时间 |
| **深度传感器** | 水压传感器（如 MS5837） | I2C 接口，精度满足定深控制需求 |
| **漏水检测** | 导电式漏水检测模块 x2 | 电控舱和电池舱各一处更稳妥 |
| **通信** | 线控 UART（串口直连） | 水下试验验证线长是否稳定，信号强度不够再考虑加驱动 |

#### 💡 JY61P vs MPU6050 选型说明

| 对比项 | MPU6050 | JY61P（推荐） |
|--------|---------|--------------|
| 输出数据 | 原始加速度+角速度，需自行姿态解算 | 直接输出欧拉角（俯仰/横滚/偏航） |
| 开发难度 | 需要 I2C 驱动 + DMP 库配置 | 串口读取，即插即用 |
| 稳定性 | 裸片温漂较明显，需额外滤波 | 内置滤波和温漂补偿 |
| 价格 | 几元 | 几十元 |
| 阶段A适用性 | ❌ 会占用较多调试时间 | ✅ 省下精力搞机械和舵机控制 |

**结论：** 阶段A以快速验证原型为目标，推荐 JY61P。阶段B想深入玩再回头研究 6050 的 DMP 也不迟。

### 7.3 软件与控制

| 软件模块 | 功能 |
|----------|------|
| **舵机初始化** | 舵机回中、限位设置、运动范围标定 |
| **尾鳍摆动控制** | 周期性摆动信号，控制摆幅和频率 |
| **方向控制** | 将左转、右转、修正航向等指令转为舵机角度 |

| **通信协议** | 接收控制指令，返回状态、电量、深度、漏水等数据 |
| **安全保护** | 通信丢失、漏水、低电压或舵机异常时进入保护状态 |
| **任务状态机** | 下水、自检、巡游、接近目标、打卡、返航等状态切换 |

## 💧 八、沉浮控制方案详解

### 阶段A推荐方案：中性浮力 + 鳍面控制升降

这是项目书已收录的方案，补充机械原理说明：

**1️⃣ 调浮力到接近中性**
- 通过配重块和浮力材料（如闭孔泡沫、中空密封腔）的配合
- 让鱼体在水中几乎不沉不浮，处于悬浮的临界状态
- 这一步靠「静水配重测试」反复微调完成

**2️⃣ 背鳍和臀鳍控制升降（类似飞机俯仰舵面原理）**
- **背鳍向上摆动 → 产生向下的水动力 → 鱼头微俯 → 下潜**
- **背鳍向下摆动 / 臀鳍调整仰角 → 产生升力 → 鱼头微仰 → 上浮**
- 鱼需要保持一定前进速度，鳍面才有足够水流产生升力/下压力
- 这就是为什么该方案「低速/原地定深能力较弱」

**3️⃣ 俯仰和横滚姿态通过 IMU（JY61P）实时反馈**
- 当检测到鱼体横滚偏差，调整单侧鳍片角度补偿
- 闭合控制回路可以大幅提升稳定性

### 方案对比

| 方案 | 优点 | 缺点 | 适用阶段 |
|------|------|------|:--------:|
| **中性浮力 + 鳍面控制升降** ✅ | 结构简单，适合首版快速实现 | 需要前进速度，低速/原地定深能力较弱 | **阶段 A** |
| **可移动配重** | 有助于调整俯仰和重心 | 响应较慢，机构占空间 | 阶段 A/B |
| **活塞式浮力舱** | 能实现更真实的沉浮控制 | 机械和防水复杂度高 | 阶段 B |
| **微型水泵调节浮力舱** | 可实现主动浮力变化 | 需处理水泵、管路、密封和响应速度 | 阶段 B |

## 🎯 阶段A主控选型：STM32F103ZET6 vs C8T6

### 引脚资源对比

| 资源项 | C8T6（48脚） | ZET6（100脚） | 阶段A需求 |
|--------|:------------:|:-------------:|:---------:|
| GPIO | ~37 | ~80 | 需驱动多个传感器+指示灯 |
| 定时器（PWM通道） | 4个 | 8个 | 至少需 3~4 路舵机 PWM |
| USART | 2 | 5 | JY61P(1路)+线控通信(1路)+调试(预留) |
| I2C | 1 | 2 | 深度传感器(1路)+预留 |
| ADC | 2通道 | 16通道 | 电池电量监测等 |
| 扩展余量 | 紧张 | 充裕 | ZET6从容应对后期调整 |

### 建议方案

| 维度 | C8T6 | ZET6 |
|------|:----:|:----:|
| 阶段A可行性 | ✅ 勉强够用，但余量很少 | ✅✅ 完全充裕 |
| 引脚紧张度 | 加蜂鸣器/LED都可能要硬挤 | 随便接，布线灵活 |
| 后续扩展空间 | 几乎为零 | 留有充足余量 |
| 首版建议 | ❌ 不推荐（遇到问题难加修） | ✅ 推荐使用 |

**结论：阶段A推荐 ZET6 做主控**，C8T6 可留作上位机调试转发或后期协处理器。等阶段B如果需要更强算力，再按需引入 ESP32-S3 等。

## ⚠️ 十一、关键风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 水下通信不稳定 | 无法可靠控制 | 首版直接用 UART 串口线控，先验证短距离稳定性 |
| 防水失败 | 电路短路、样机损坏 | 分阶段浸水测试，电控舱独立密封，线缆出口重点处理 |
| 重心与浮心不匹配 | 鱼体侧翻或游动摇晃 | 可调配重，先静水调平再动态测试 |
| 舵机力矩不足 | 推进弱或鳍片卡滞 | 先做单鳍负载测试，预留更大扭矩舵机安装空间 |
| 传动机构卡滞 | 舵机发热、动作失控 | 简单可靠的连杆/曲柄机构，减少复杂齿轮级数 |
| PETG 外壳渗水 | 内部进水或结构变形 | 增加壁厚，优化打印参数，外表面做涂层或密封处理 |
| 沉浮控制不足 | 无法到达不同高度目标点 | 首版调中性浮力，必要时升级主动浮力舱 |
| 控制算法耦合 | 转向、俯仰、沉浮互相影响 | 先手动标定各鳍片作用，再逐步加入闭环控制 |

## ✅ 十二、验收标准

首版样机建议按以下标准验收：

- **防水：** 静置浸水 30 分钟后，电控舱和电池舱无明显进水
- **上电：** 入水前自检通过，舵机可回中，通信稳定
- **姿态：** 静水中鱼体保持基本水平，不持续侧翻
- **推进：** 尾鳍摆动后能连续前进，舵机无明显过热
- **转向：** 能完成可控左转、右转和航向修正
- **沉浮：** 能完成上浮、下潜或不同深度目标接近
- **打卡：** 能按比赛或演示规则完成至少一次目标点打卡
- **维护：** 电池、主控、配重可在不破坏主体结构的情况下拆装调整
