# 🐟 仿生鱼项目计划书

> **版本：** v3.2 ｜ **日期：** 2026-05-31 ｜ **状态：** 主控改为 STM32F103C8T6，ESP32 作为上位机

---

## 📌 一、项目定位

制作一只以**稳定性和可控性**为优先目标的**仿生虎鲸**，用于水下任务演示或比赛。要求：

- 水中稳定游动
- 可控沉浮
- 通过尾鳍（上下摆动推进）、背鳍与尾段扭腰联动（转向）、腹鳍（姿态辅助）配合完成方向控制
- 实现对不同高度、不同位置坐标点的接近与打卡

**首版策略：** STM32F103C8T6 控制 + ESP32 上位机，先做可靠无线遥控原型 → 再决定是否加入视觉识别与自主决策

---

## 🎯 二、项目目标

### 2.1 核心目标

| 序号  | 目标        | 说明                   |
| :-: | --------- | -------------------- |
|  1  | 制作可下水实体样机 | 能稳定运行                |
|  2  | 基础推进      | 防水电机 + 曲柄摇杆驱动尾鳍上下摆动  |
|  3  | 方向控制      | 背鳍舵机通过齿轮组联动尾段扭腰，实现转向 |
|  4  | 姿态辅助      | 左右腹鳍独立舵机，调节横滚和辅助沉浮   |
|  5  | 可控沉浮      | 上浮、下潜、定深控制           |
|  6  | 任务打卡      | 到达不同坐标点并完成触发/接近/识别   |
|  7  | 水下安全      | 电路、电池、电机、舵机区域可靠防水    |

### 2.2 首版边界

**采用「无线遥控 + 半自动控制」方式验证原型。**

水下 2.4G 无线信号衰减明显，本版采用 **ESP32 无线中继方案**：
- 2.4G 天线延伸至背鳍内部，水面附近通信稳定
- ESP32 作为通信转发层，通过 UART 与 STM32 主控交互
- 上位机（遥控器/手机/PC）通过 2.4G 与 ESP32 通信

后期增强版可加入摄像头、视觉识别模块、更强算力主控或协处理器，实现自动寻点、自动接近和自动打卡。

## 🔄 三、典型任务流程

```
下水前检查（防水/电量/电机舵机回中/通信/传感器）
       ↓
入水姿态稳定（无侧翻、漏水、失控）
       ↓
操作者控制前进、转向、上浮、下潜
       ↓
接近指定坐标点 → 调整深度
       ↓
到达后完成打卡动作（触碰/停留/识别/触发传感器）
       ↓
切换至下一个目标点
       ↓
任务结束 → 返航或人工回收 → 记录数据
```

## 📋 四、需求拆解

### 4.1 功能需求

| 编号 | 需求 | 优先级 | 说明 | 验收方式 |
|:----:|------|:------:|------|----------|
| F01 | 水中稳定漂浮与姿态保持 | **P0** | 入水后不应明显侧翻、头重脚轻或持续翻滚 | 静水中观察姿态，记录横滚和俯仰偏差 |
| F02 | 尾鳍推进 | **P0** | 防水电机驱动尾鳍上下摆动，提供前进动力 | 在水槽或泳池中完成连续前进 |
| F03 | 方向控制 | **P0** | 背鳍舵机通过齿轮组联动尾段扭腰实现转向 | 能完成左转、右转、修正航向 |
| F04 | 姿态辅助 | **P0** | 腹鳍舵机辅助横滚修正和沉浮微调 | 能配合转向和沉浮需求调整姿态 |
| F05 | 可控沉浮 | **P0** | 根据目标点高度完成上浮、下潜或定深 | 在不同深度位置完成稳定运动 |
| F06 | 坐标点打卡 | **P0** | 能接近或触发比赛要求的坐标点 | 按比赛规则完成打卡动作 |
| F07 | 无线通信 | **P0** | 通过 2.4G 无线链路传输控制指令与调试数据 | 水面附近通信稳定，指令响应可靠 |
| F08 | 电机/舵机联动 | **P0** | 曲柄摇杆机构顺畅；齿轮组联动背鳍与尾腰无卡滞 | 运动机构无干涉、无过载发热 |
| F09 | 防水与漏电保护 | **P0** | 主控、电池、电机、舵机接线处需要密封 | 浸水测试后内部无水、无短路 |
| F10 | 配重与重心调节 | **P0** | 内部设置可调配重，降低重心并改善平衡 | 调整配重后姿态明显改善 |
| F11 | 视觉自主寻点 | **P2** | 增强版能力，需要摄像头和更强算力 | 完成目标识别和自主接近测试 |

### 4.2 非功能需求

| 类型 | 要求 | 说明 |
|------|------|------|
| **稳定性** | 首版优先稳定，不追求高速 | 虎鲸低重心圆润体形，避免过窄、过高、重心偏上结构 |
| **可维护性** | 电机、舵机、电池、主控、配重方便拆装 | 预留检修口，避免每次维护破坏密封结构 |
| **防水性** | 电控舱、线缆出口、电机/舵机区域重点防水 | 密封圈、密封胶、轴封、防水接头或独立密封舱 |
| **可制造性** | 外壳使用 3D 打印 PETG 材质 | 兼顾强度、外观、打印难度和后处理防水 |
| **可调试性** | 保留日志、串口或上位机调试接口 | 便于定位电机、舵机、姿态、深度和通信问题 |
| **安全性** | 电池、电路和电机需过流保护 | 防止水下短路、发热和失控 |

## 🏗️ 五、总体技术路线

### 阶段 A：可靠无线原型（当前阶段）

目标是尽快做出可下水、可游、可控方向与深度的样机。

```
尺寸：体长 50~60cm（仿生虎鲸）
推进：防水电机 × 1 + 曲柄摇杆机构 → 尾鳍上下摆动
转向：舵机 × 1 + 齿轮组 → 联动背鳍与尾段扭腰
姿态：舵机 × 2 → 左右腹鳍独立控制
主控：STM32F103C8T6（运动控制 + 传感器采集）
通信中继：ESP32（带 IPEX 接口，2.4G 无线接收转发）
天线：IPEX 同轴电缆延伸至背鳍内部 + 2.4G PCB/陶瓷天线
外壳：PETG 3D 打印
配重：配重 + 浮力材料调节重心与浮心
控制：先完成人工遥控打卡，再优化控制算法
上位机：遥控器/手机/PC
```

### 阶段 B：自主增强版本

在阶段 A 稳定后加入视觉/传感器自主能力。

```
摄像头/视觉模块
更强算力 MCU 或协处理器
目标识别、自动寻点、自动打卡
深度闭环、姿态闭环、任务状态机
```

## 🧩 七、模块设计要求

### 7.1 机械结构

| 模块 | 设计要求 | 需要完成的工作 |
|------|----------|---------------|
| **鱼体外壳** | 仿生虎鲸构型，体长 50-60cm；PETG 3D 打印 | CAD 外形，确定长宽高、壁厚、分段方式和检修口位置 |
| **尾鳍推进机构** | 防水电机 + 曲柄摇杆机构实现尾鳍上下摆动 | 设计电机座、曲柄摇杆尺寸，确定摆动幅度和频率范围 |
| **转向机构** | 1×舵机 + 齿轮组，联动背鳍和尾段扭腰 | 设计齿轮减速比、输出轴位置，确保背鳍和尾腰同步转动 |
| **腹鳍** | 2×舵机独立驱动左右腹鳍，用于姿态辅助与沉浮微调 | 确定腹鳍位置、面积、舵机安装方式和摆角范围 |
| **传动机构** | 尾鳍曲柄摇杆需顺畅无卡滞；齿轮组合理传动比 | 做运动仿真或手动样件验证，避免干涉和过载 |
| **防水结构** | 主控、电池、电机、舵机接线处重点密封 | 密封舱、密封圈槽、线缆防水接头、旋转轴轴封 |
| **配重结构** | 内部可调配重，降低重心并抑制游动摇晃 | 可移动配重槽，支持前后、左右、上下微调 |
| **背鳍天线舱** | 背鳍内部预留空腔，安装 2.4G 天线 | IPEX 同轴线沿背部内壁走线，天线固定座 |

### 7.2 电控系统

| 模块 | 建议方案 | 说明 |
|------|----------|------|
| **主控** | STM32F103ZET6（已有 C8T6 备选） | 专管电机PWM调速 + 舵机PWM控制 + 传感器数据采集 |
| **上位机（通信中继）** | ESP32（带 IPEX 接口版本） | 接收遥控/手机/PC指令，UART 透传至 STM32 C8T6 |
| **天线** | IPEX → SMA 转接线 + 2.4G PCB/陶瓷天线 | 同轴线沿背部内壁走至背鳍，固定于内部空腔 |
| **防水电机** | 防水直流减速电机（需选型，匹配扭矩与转速） | 驱动尾鳍上下摆动，配合曲柄摇杆机构 |
| **电机驱动** | 直流电机驱动模块（支持PWM调速和正反转） | MOS管H桥驱动，注意电流余量 |
| **转向舵机** | 防水舵机（如 DS3218MG）× 1 | 背鳍联动尾段扭腰，齿轮组传动 |
| **腹鳍舵机** | 防水舵机（如 DS3218MG）× 2 | 左右腹鳍独立控制 |
| **电源** | 2S 锂电 7.4V + 稳压模块 + 保险保护 | 电机和舵机建议独立供电或分级稳压 |
| **姿态传感器** | JY61P（IMU） | 检测俯仰、横滚、偏航趋势 |
| **深度传感器** | MS5837（水压传感器） | 定深和沉浮控制 |


### 7.2b C8T6 引脚资源评估

| 资源类型 | STM32F103C8T6 可用数量 | 本项目需求 | 余量 |
|---------|---------------------|-----------|:--:|
| **GPIO** | 37 个 | 约 18~22 个（舵机PWM×3、电机PWM×2、UART×4、I2C×2、SWD×2、LED/按键×3~5） | **充足** |
| **Timer PWM 通道** | TIM1×4 + TIM2×4 + TIM3×4 + TIM4×4 = 16 路 | 舵机 3 路 + 电机 H桥 2 路 = 5 路 | **充裕** |
| **UART** | USART1 / USART2 / USART3 = 3 个 | ESP32 通信 ×1 + 调试串口 ×1 = 2 个 | 1 个备用 |
| **I2C** | I2C1 ×1 | JY61P + MS5837 共享总线 = 1 路 | **刚好** |
| **SPI** | SPI1 ×1 | 暂不使用（为后续扩展预留） | 1 个备用 |
| **Flash** | 64 KB | 预计 20~30 KB（FreeRTOS + 驱动 + 控制逻辑） | **充足** |
| **RAM** | 20 KB | 预计 8~12 KB | **充足** |

> ⚠️ **注意：** C8T6 无 FSMC（无法外扩 SRAM/Flash）、无 USB（如需 USB 调试需额外转接）、无 CAN。对于阶段 A 的纯控制需求无影响，但阶段 B 如加入视觉处理则需要增强算力（如额外添加协处理器）。

### 7.3 软件与控制

| 软件模块 | 功能 |
|----------|------|
| **ESP32 无线透传** | 2.4G 接收上位机指令 → UART 转发给 STM32；STM32 回传状态 → 无线发回上位机 |
| **电机初始化** | 电机回中、频率/幅度参数标定 |
| **电机推进控制** | PWM 调速控制尾鳍摆动频率，调幅控制摆幅 |
| **转向控制** | 背鳍舵机角度控制，齿轮组联动尾段扭腰 |
| **腹鳍控制** | 左右腹鳍独立角度控制，辅助横滚修正和沉浮微调 |
| **沉浮控制** | 根据深度和姿态传感器综合调节腹鳍与整体姿态 |
| **姿态稳定** | 根据 IMU 数据修正横滚和俯仰 |
| **通信协议** | 定义上位机 ↔ ESP32 ↔ STM32 之间的指令帧格式 |
| **安全保护** | 通信丢失、低电压或执行器异常时进入保护状态 |
| **任务状态机** | 下水、自检、巡游、接近目标、打卡、返航等状态切换 |

## 💧 八、沉浮控制方案选择

### 采用课程中提到的仿生鲨鱼法

可控沉浮是项目关键难点之一。通过调节中性复利+舵面朝向控制pitch角

| 方案                  | 优点            | 缺点                 |   适用阶段   |
| ------------------- | ------------- | ------------------ | :------: |
| **中性浮力 + 鳍面控制升降** ✅ | 结构简单，适合首版快速实现 | 需要前进速度，低速/原地定深能力较弱 | **阶段 A** |
| **可移动配重**           | 有助于调整俯仰和重心    | 响应较慢，机构占空间         |  阶段 A/B  |
| **活塞式浮力舱**          | 能实现更真实的沉浮控制   | 机械和防水复杂度高          |   阶段 B   |
| **微型水泵调节浮力舱**       | 可实现主动浮力变化     | 需处理水泵、管路、密封和响应速度   |   阶段 B   |

**首版建议：** 先把鱼体调到接近中性浮力，再用腹鳍（及背鳍辅助）控制上浮和下潜。如果比赛要求低速定深或原地悬停，再升级为主动浮力调节。

## ⚠️ 十一、关键风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 水下无线通信衰减 | 信号丢失、控制不稳定 | ESP32 中继 + 天线延伸至背鳍；水面附近通信；必要时降级为浮标中继 |
| 2.4G 同轴线阻抗不匹配 | 信号质量差、辐射效率低 | 使用标准 50Ω IPEX 转接线，不自行使用普通导线延长 |
| 防水电机进水 | 电机短路、推进失效 | 选用正规防水电机，轴封和接线处做额外密封处理 |
| 曲柄摇杆机构卡滞 | 推进失效或电机过载 | 运动仿真验证，打印样件手动测试后再装机 |
| 齿轮组联动背鳍/尾腰 | 转向失效或舵机堵转 | 合理的齿轮侧隙和润滑，预留调节余量 |
| 防水失败 | 电路短路、样机损坏 | 分阶段浸水测试，电控舱独立密封，线缆出口重点处理 |
| 重心与浮心不匹配 | 鱼体侧翻或游动摇晃 | 可调配重，先静水调平再动态测试 |
| 电机力矩不足 | 推进弱、无法克服水流 | 按鱼体阻力和目标速度估算所需扭矩后选型 |
| 舵机力矩不足 | 转向/腹鳍控制弱、过载发热 | 选 DS3218MG 级防水舵机，预留安装空间 |
| PETG 外壳渗水 | 内部进水或结构变形 | 增加壁厚，优化打印参数，外表面做涂层或密封处理 |
| 沉浮控制不足 | 无法到达不同高度目标点 | 首版调中性浮力，必要时升级主动浮力舱 |
| 控制算法耦合 | 转向、俯仰、沉浮互相影响 | 先手动标定各鳍片作用，再逐步加入闭环控制 |
| ESP32 重启导致通信中断 | 临时失控 | 增加看门狗定时器，STM32 侧设超时保护自动进入安全状态 |

## ✅ 十二、验收标准

首版样机建议按以下标准验收：

- **防水：** 静置浸水 30 分钟后，电控舱和电池舱无明显进水
- **上电：** 入水前自检通过，电机/舵机可回中，通信稳定
- **姿态：** 静水中鱼体保持基本水平，不持续侧翻
- **推进：** 尾鳍上下摆动后能连续前进，电机无明显示过热
- **转向：** 背鳍联动尾腰，能完成可控左转、右转和航向修正
- **腹鳍：** 左右腹鳍可独立控制，辅助横滚修正和沉浮微调
- **沉浮：** 能完成上浮、下潜或不同深度目标接近
- **打卡：** 能按比赛或演示规则完成至少一次目标点打卡
- **维护：** 电池、主控、配重可在不破坏主体结构的情况下拆装调整

> **总结：** 本版目标：体长 50-60cm 的**仿生虎鲸**。
>
> **推进** → 防水电机 + 曲柄摇杆驱动尾鳍上下摆动
>
> **转向** → 1×舵机 + 齿轮组联动背鳍和尾段扭腰
>
> **姿态** → 2×舵机独立控制左右腹鳍
>
> **通信** → ESP32 无线中继 + 背鳍天线（IPEX）
>
> 只要尾鳍能稳定上下摆推进、背鳍联动尾腰可靠转向、腹鳍辅助沉浮和姿态，就解决了最大的不确定性。后续视觉和自主能力作为第二阶段增强。
